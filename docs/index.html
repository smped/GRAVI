Description

This is a standardised workflow for beginning the comparison between two or ChIP targets, using BAM files as the starting point. Treatment groups and targets are specified using config/config.yml.

Workflow

here::here("workflow", "rules", "rulegraph.dot") %>%
  readLines() %>%
  rm_dot_node(node = "\"all\"") %>%

  grViz()

Differential Binding

The specified comparisons are given below, where ‘Vs.’ can be interpreted as a ‘-’ sign. This effectively makes the first listed group as the treatment and the second listed as the reference or control. Differential binding analysis will be performed for every ChIP target where both treatment groups are present.

  • DHT Vs. Veh

Pairwise Comparisons

All combinations of the comparisons defined above will will be used to perform pairwise comparisions.

Settings

Genome

  • The genome build (GRCh37) was sourced from Gencode release 33
  • Blacklisted sites were sourced from https://github.com/boyle-lab/Blacklist/ for this specific genome build

External Data

The external data was:

  • rnaseq: PDX_HCI-005_Veh_DHT_Sarm_RNASeq.tsv
  • features: h3k27ac_features.gtf
  • hic: encode_hic.bedpe
  • coverage: H3K27Ac_Veh.bw and H3K27Ac_DHT.bw

Peak Detection

Macs2 callpeak settings were default with the following exceptions:

  • gsize: hs
  • fdr: 0.05
  • keep_duplicates: all

Two additional QC settings were incorporated for peak filtering.

  • For a sample to pass initial QC, at least 10% of the peaks identified in the sample identifying the highest number of peaks need to be returned. Any many instances, no peaks will be returned in any condition and whilst in this situation all samples will be retained. However, samples significantly diverging from the ‘best’ sample will be excluded.
  • For a peak to be included in the set of “Oracle Peaks”, it needs to overlap an identified peak in at least 30% of samples passing the above QC steps
